Definición de las capas del AUTOENCODER¶

Para el AUTOENCODER se definen las siguientes capas:

  • Una capa de entrada Dense con 784 neuronas para la representacin alternativa de los datos en 1D y funcin de activacin relu.
  • Una primera capa de codificacin Dense con 256 neuronas a la representacin 1D de los datos.
  • Una segunda capa de codificacin Dense con 128 neuronas a la representacin 1D de los datos.
  • Una capa intermedia de hallazgos/aprendizaje Dense con 64 neuronas a la representacin 1D de los datos.
  • Una primera capa de decodificacin Dense con 128 neuronas a la representacin 1D de los datos.
  • Una primera capa de decodificacin Dense con 256 neuronas a la representacin 1D de los datos.
  • Una capa de salida Dense con 784 neuronas para la representacin alternativa de los datos en 1D y funcin de activacin relu.
  • Unas capas Dropout entre cada capa de aprendizaje (codificacin y decodificacin) con 20% de desactivacin de los datos de entrada para evitar un sobre ajuste del modelo.
  • Todas las capas de aprendizaje (codificacin, decodificacin e intermania) tiene una funcion de activacin relu para evitar la saturacin de la salida de las capas.

layerList = (
    # capa de entrada
    Dense(neurons, input_shape = inshape, activation = inn, name = "in_layer"),
    Dropout(ldrop, seed = 42),

    # primera capa de codificacion neurons = 256
    Dense(2**int(neurons**(0.5)-20), input_shape = lshape, activation = act),
    Dropout(ldrop, seed = 42),

    # segunda capa de codificacion neurons = 128
    Dense(2**int(neurons**(0.5)-21), input_shape = lshape, activation = act),
    Dropout(ldrop, seed = 42),
    
    # capa intermedia neurons = 64
    Dense(2**int(neurons**(0.5)-22), input_shape = lshape, activation = act, name = "mid_layer"),
    Dropout(ldrop, seed = 42),

    # primera capa de codificacion neurons = 128
    Dense(2**int(neurons**(0.5)-21), input_shape = lshape, activation = act),
    Dropout(ldrop, seed = 42),
    
    # segunda capa de decodificacion neurons = 256
    Dense(2**int(neurons**(0.5)-20), input_shape = lshape, activation = act),
    Dropout(ldrop, seed = 42),

    # capa de salida
    Dense(output, input_shape = outshape, activation = out, name = "out_layer"),
)

Definición de las capas MLP¶

Para el MLP se definen las siguientes capas:

  • Una capa de entrada Flatten que recibe la imagen 2D y lo transforma a un arreglo 1D para facilidad de procesamiento.
  • Una capa de 80 neuronas Dense como capa de entrada para los datos con representacin 1D.
  • Una capa Dropout con 20% de desactivacin de los datos de entrada para evitar un sobre ajuste del modelo, se recalca que entre cada capa de trabajo se utiliza Dropout con el mismo propsito.
  • Una capa de 60 neuronas Dense como capa como primera capa de aprendizaje.
  • Una capa de 40 neuronas Dense como capa como segunda capa de aprendizaje.
  • Todas las capas de aprendizaje tienen funcin de activacin relu para evitar la explosin/desvanecimiento de los gradientes.
  • Una capa con 10 neuronas Dense como capa de salida para clasificacin con funcin de activacin softmax para reconocer categoras por separado.

Optimización de Hiper-parámetros¶

para optimizar el desempeo del Perceptrn se utiliza la librera Talos porque es relativamente fcil de crear un modelo optimizable e iterable en el espacio de hiper-parmetros definiendo una funcin con parmetros especficos. Para lograr esto se realizan los siguientes pasos:

  • Se define una funcin como modelo para el modelo de optimizacin Talos.
  • Se define un espacio de bsqueda de hiper-parmetros del MLP.
  • Se ejecuta una bsqueda aleatoria (opcin por defecto de Talos) en una fraccin reducida del espacio de hiper-parmetros del MLP.
  • Despus, se analizan los resultados para obtener la mejor versin del modelo entrenado y se resume los resultados del modelo seleccionado durante el proceso de optimizacin.

# redefiniendo el modelo preliminar en funcion para optimizar
def fashionPerceptron(X_train, y_train, X_test, y_test, params):
  
    inshape = X_train[0].shape
    lshape = np.reshape(X_train[0], -1).shape
    ver = 0
    
    # definiendo el modelo MLP de Keras
    model = Sequential()
    
    # capa para representacion alternativa de (n,m) a (m*m)
    model.add(Flatten(input_shape = X_train[0].shape))

    # capa de entrada para el modelo
    model.add(Dense(params["first_neuron"], input_shape = inshape, activation = params["activation"]))
    
    # paso intermedio para hacer dropout yevitar overfit
    model.add(Dropout(params["dropout"]))
    
    # capas ocultas del modelo
    for i in range(params['hidden_layers']):
        model.add(Dense(params['hidden_neuron'], input_shape = lshape, activation = params["activation"])),
        model.add(Dropout(params['dropout']))
    
    # capa de salida del modelo
    model.add(Dense(params["last_neuron"], input_shape = lshape, activation = params["last_activation"]))
    
    # compilando las condiciones de optimizacion y ajuste del MLP Keras
    model.compile(optimizer = params["optimizer"], loss = params["losses"], metrics = ["acc"])

    # entrenando el modelo MLP Keras
    history = model.fit(
        x = np.array(X_train), 
        y = to_categorical(np.array(y_train), params["last_neuron"]), 
        epochs = params["epochs"], verbose = ver, 
        workers = 8, use_multiprocessing = True,
        batch_size = params["batch_size"],
        callbacks = [talos.utils.ExperimentLogCallback("fashionPerceptronExp", params)],
        validation_data = (np.array(X_test), to_categorical(np.array(y_test), params["last_neuron"]))
    )
    
    return history, model

Conclusiones¶

Se utiliza la estructura DataFame para guardar todas las representaciones de datos necesarias durante el ejercicio. Aunque, se establece la perdida y la precisin como medidas del rendimiento de los modelos, en el Autoencoder la precisin no es significativa y se considera tomar otra medida/mtodo como minimizar la prdida del modelo para poder agilizar la experimentacin.

La grafica de la curva de aprendizaje en este caso es irrelevante si se observa la precisin/perdida juntas, solo al momento de reconstruir las imgenes y desplegar la representacin condensada de los datos se puede ver que el modelo realmente est funcionando.

Como en la primera parte del taller, se observa que el modelo tiene un buen comportamiento global, pero se observa un menor rendimiento en las categoras 0 (T-shirt/top/camiseta), 2 (Pullover/buso), 4 (Coat/chaqueta) y 6 (Shirt/camisa). Esto tiene que ver con que las clases sean parecidas entre s, y aunque la representacin condensada ayuda un poco, esta tambin se confunde porque varias caractersticas/atributos significativos son iguales entre clases. Como, por ejemplo: el largo de las mangas, el tipo de cuello, entre otros.

En la optimizacin de hiper-parmetros se utiliza la librera por extensin Talos con la misma metodologa que el primer punto con una bsqueda aleatoria de los mejores hiper-parmetros en el espacio. De nuevo, la precisin se selecciona como parmetro objetivo a maximizar y se usa una poblacin de mximo 3%. Por ltimo, se configura la bsqueda para minimizar el uso de parmetros/hiper-parmetros no significativos (correlacionados linealmente) con la precisin del modelo. Al reducir el numero de neuronas y simplificar el modelo se observ una mejora significativa en el tiempo de optimizacin.

El modelo final optimizado es un MLP con:

  • Una capa de entrada Flatten que recibe la imagen 2D y la transforma a un arreglo 1D para su procesamiento.
  • Una capa de 60 neuronas Dense como primera capa de aprendizaje y activacin tipo relu
  • Entre capas de trabajo Dropout con 10% de desactivacin para evitar sobreajustes.
  • Una capa de 40 neuronas Dense como capa como segunda capa de aprendizaje y activacin tipo relu.
  • Una capa con 10 neuronas Dense como capa de salida para clasificacin con funcin de activacin softmax para reconocer categoras por separado.

La bsqueda muestra una correlacin fuerte entre la precisin y la perdida de tendencia lineal, lo que coincide con la literatura. La precisin tiene un mnimo de 81% y un mximo de 83% (optimo encontrado). En promedio el rendimiento (precisin) de los modelos esta alrededor 80%. Tambin se observa una correlacin entre el tamao de la muestra, el numero de neuronas en la capa de entrada y el nmero de capas ocultas. Por el contrario, el Dropdown no afecta significativamente el modelo ni tampoco el numero de neuronas por capa. Esto ltimo se relaciona con la simplificacin de los datos con la representacin condensada. En general las soluciones son mas estables entre si, pero los modelos todava no pueden reconocer bien las categoras confusas (0, 2, 4 y 6) y no hay cambio significativo despus de la optimizacin, seguramente porque se utiliza la representacin condensada con caractersticas fundamentales de cada categora.
